React4j 0.24 released

December 11, 2017

React4j Project

Full Changelog

Changes in this release:

Fixed
  • [core] Fixed bug where ReactArezComponent.scheduleStateUpdate() could result in a call to Component.forceUpdate() within the context of state updater. This would trigger an invariant failure within ReactJs code.
Changed
  • üí• [core] Add a parameter onStateUpdateComplete to Component.scheduleStateUpdate(...) that is invoked by the react runtime after the state update has completed. Add additional overloaded variants of Component.scheduleStateUpdate(...) where the call back is not present. This had the impact of changing the variant without the onStateUpdateComplete parameter to be a final method invocation.
  • üí• [core] Remove the remaining Component.renderAs* methods as they do not provide significant benefit relative to cost now that the ReactNode class has been introduced.

React4j 0.23 released

December 10, 2017

React4j Project

Full Changelog

Changes in this release:

Changed
  • [core] Remove unused methods JsUtil.asList(..) and JsUtil.getPrototypeForClass(..).
  • [annotations] Add parameter inject to the @ReactComponent annotation that explicitly controls whether the enhanced component is generated with an @Inject annotation on the constructor. The default value for this parameter is IF_DETECTED and will generate a constructor using the existing algorithm. i.e. An @Inject annotated cosntructor will be generated if a field of method of the component is annotated with @Inject.
  • [annotations] Add parameter dagger to the @ReactComponent annotation that explicitly controls whether a dagger component is created for the react component. The default value for this parameter is IF_DETECTED and will generate a constructor using the existing algorithm. i.e. A module is created if the dagger classes are on the compile classpath.
  • üí• [core] Upgrade com.google.jsinterop:jsinterop-annotations library to version 1.0.2.
  • üí• [core] Upgrade com.google.jsinterop:base library to version 1.0.0-beta-3.
  • üí• [core] Upgrade com.google.elemental2:* libraries to version 1.0.0-beta-3.
  • üí• [core] Upgrade Arez library to version 0.34.
  • üí• [arez] Skip the shouldComponentUpdate() lifecycle method when Arez re-schedules a render action for subclasses of ReactArezComponent.

React4j 0.20 released

December 1, 2017

React4j Project

Full Changelog

Changes in this release:

Fixed
  • [arez] Enhance ReactArezComponent so that autoruns are not scheduled when the component is created but are instead scheduled after the componentDidConstruct lifecycle method was invoked. This ensures that @Autorun methods are not invoked when the component is invalid (i.e. when the native component has not been attached or when the componentDidConstruct method has yet to complete.)
Changed
  • [arez] Upgrade Arez to version 0.32.
  • Upgrade Buildr to version 1.5.4.
  • [processor] Detect the presence of the @Inject annotation on fields or methods of classes annotated with @ReactComponent. If detected then ensure that the enhanced component either; sets the inject = true parameter on the @ArezComponent annotation for arez components or defines a default constructor with an @Inject annotation for non-arez components. This is the first step to enabling integration of React4j components with injection frameworks such as Dagger2 or Google GIN.
